# S8-80 — Manual Verification Checklist Persistence and Re-Display

**Ticket:** S8-80
**Scope:** Add documentation and mechanical test coverage for the QA manual-verification-checklist persistence + re-display behaviour introduced by commits `e74a152`, `3e729fa`, `e5e2a9a` (all `Refs: S8-80`) across the existing `QA-TESTS/` three-tier pyramid.
**Owner:** TBD
**Status:** Plan — not yet started

---

## Goal

The S8-80 commits make the QA manual verification checklist a **persisted, verbatim, always-re-displayed artefact**. Specifically:

1. `code-reviewer` Call B writes the checklist to `generated-docs/qa/epic-N-[slug]/story-M-[slug]-verification-checklist.md`.
2. On every "Issues found" fix cycle, the orchestrator re-reads that file and includes the **complete checklist** in the next `NEEDS_APPROVAL` payload — copied verbatim, no abbreviation, no "see above".
3. On free-text user replies (questions instead of option picks), the orchestrator answers, then re-asks `AskUserQuestion` with the full checklist re-read from the same file.

`HOW-IT-WORKS.md` line 282–283 already mentions persistence in passing. What is missing is:

- An end-to-end behavioural description of the re-display contract (first ask → fix cycle → free-text reply → option pick).
- A Tier 3 manual test that exercises both re-display paths.
- Reproducible inputs that drive the fix cycle and the free-text branch.
- Strategy entries that map the new mechanical tests into the pyramid.
- Implementation of the Tier 1 (structural) and Tier 2 (log-replay) tests.

Two stages in order: **docs first, tests second.** Within each, files are sequenced so each one consumes the previous one's outputs.
